# How APIs Work in Digital Banking: The Invisible Connection Behind Every Transaction

## Introduction
When we open a mobile banking app, check our balance, send money, or pay a bill, it feels like everything happens inside one application.  
Behind the scenes, multiple systems are talking to each other — and APIs make that possible.

## What is an API?
An **Application Programming Interface (API)** is a way for different software systems to communicate.  
Think of it like a waiter in a restaurant:
- Customer = Mobile banking app  
- Waiter = API  
- Kitchen = Banking systems/databases  

The app doesn’t directly touch the bank’s database — it sends requests through the API.

## Where APIs Appear in Digital Banking
- **Balance checks** → API validates request, fetches data, returns balance.  
- **Money transfers** → API coordinates authentication, account validation, fraud checks, and transaction confirmation.  
- **Payments & integrations** → APIs connect banks with merchants, fintechs, and payment providers.  
- **Open banking** → Customers securely share data with other financial services.  

## When APIs Fail
A “Transaction failed” message could mean issues in:
- Mobile app  
- API gateway  
- Network connection  
- Authentication service  
- Core banking system  
- Database  

Understanding APIs helps support teams trace failures quickly.
